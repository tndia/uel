# Security Policy

## Reporting a Vulnerability

Se você encontrar uma vulnerabilidade (ex.: XSS, vazamento de dados), por favor **não** abra uma issue pública com detalhes exploráveis.

Abra uma issue com descrição **limitada** (sem PoC agressivo) e solicite um canal privado, ou use Discussions com informações reduzidas.

## Supported Versions

A versão publicada no GitHub Pages é a versão suportada.
